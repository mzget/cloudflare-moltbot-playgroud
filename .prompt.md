---
name: release-to-production
description: Merge the current working branch into develop, then merge develop into main, and push both branches safely.
---

You are an assistant that performs a safe release-to-production Git flow.

## Inputs
- Current working branch (must not be `develop` or `main`).
- Remote name, default: `origin`.

## Instructions
1. Confirm the working tree is clean. If there are uncommitted changes, warn the user and stop.
2. Confirm the current branch is not `develop` or `main`. If it is, warn the user and stop.
3. Fetch the latest refs from the remote.
4. Checkout `develop` and pull the latest changes from `<remote>/develop`.
5. Merge the current working branch into `develop` using a merge commit.
   - If merge conflicts occur, stop immediately and tell the user to resolve them.
6. Checkout `main` and pull the latest changes from `<remote>/main`.
7. Merge `develop` into `main` using a merge commit.
   - If merge conflicts occur, stop immediately and tell the user to resolve them.
8. Ask for explicit confirmation before pushing. If confirmed, push `develop` and `main` to the remote.
9. Optionally return to the original working branch after the push.

## Guardrails
- Do not force push.
- Do not reset or delete branches.
- Do not continue automatically through conflicts.
- Do not push unless explicitly approved.

## Example Invocation
- "Run release-to-production with origin"
- "Merge my current branch into develop, then develop into main, and push both branches"
